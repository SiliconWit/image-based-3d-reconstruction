## Overview
- Considering the reconstruction of a target object, the goal of the NBV algorithm is to provide optimal poses from which images to be used for reconstruction are to be taken from given a single reference RGB image of the target.
- The algorithm has 2 parts to it. 
	1. Identification of the Regions of Interest(RoIs). These are regions of the object that stand out relative to the rest of the object. They are derived from a depth map computed from the RGB image.
	2. Deriving 3D poses that focus on the RoIs. The 3D poses are based on a point cloud derived from an RGBD object made up of the original RGB image and its corresponding depth map, which is computed through an AI-based Monocular Depth Estimator.

## Structure from Motion
Structure from Motion (SfM) is a photogrammetric range imaging technique that estimates three-dimensional structures from two-dimensional image sequences[WhatStructureMotion,10PhotogrammetryStructurefromMotion2021]. The process involves using overlapping images to create a 3D point cloud of an object or landscape3. It starts with feature extraction and matching, followed by geometric verification. The resulting scene graph serves as the foundation for the reconstruction stage, which seeds the model with a carefully selected two-view reconstruction, before incrementally registering new images, triangulating scene points, filtering outliers, and refining the reconstruction using bundle adjustment[10PhotogrammetryStructurefromMotion2021]. The outputs of SfM include 3D locations of points corresponding to the extracted features, as well as camera parameters which can be used in subsequent reconstruction operations. The 3D points from resulting from SfM is a reconstruction in itself, even though sparse in nature compared to what can be achieved with Multview-Stereo(MVS). 

Given multiples set of input images that have been calibrated, the SfM process is capable of producing consistent results in terms of depth and scale. The number of points reconstructed is determinant of the quality of the image taken, as well as the feature extraction algorithms adopted in the SfM pipeline.
## Translation of 3D Poses
The second part of the NBV algorithm is critical to application of the algorithm through a robotic system. While simulation of the poses within the NBV environment might be "in order", this does not always translate perfectly to the calling environment. A perfect translation of the determined poses in the calling environment assumes that RGBD-based point cloud corresponded accurately to the morphology, scale and position of the target object in the calling environment. This is however not the case as key to this is the accuracy of the depth map. The depth map is estimated using Zoe-Depth[Zoe-Depth], an AI-based monocular depth estimator that is capable of giving absolute depth. The results of the depth estimator are not accurate enough to provide the desired fidelity of the RGBD point cloud. This calls for some alternative means to the point cloud before determining the poses.

To this end, a step-by-step methodology is proposed to correct the point cloud based on the results from SfM. The solution relies on the consistency and speed of structure from motion in producing the sparse point cloud. The sparse point cloud can serve as a ground truth in "calibrating" the RGBD-based(target) point cloud. Calibration in this case merely refers to transforming the target point cloud to match the scale, location and orientation of the target object in the calling environment, considering that the sparse point cloud from SfM is the ground truth. Below is an outline of the calibration pipelines:
	1. Capture Preliminary Images for SfM
	2. Initiate Structure from Motion on the 5 images
	3. Clean the Sparse Point Cloud
	4. Extract Transformation Properties
### Capture Preliminary Images for SfM

The number of images chosen for this purpose should be just enough to allow for enough points for the adequate points for the sparse point clouds but at a relatively shorter time as the process is done online. 5 Points were chosen, 4 of them making up the vertices of a "square"; the adjacent points are subtended to one another by an arc of 24 degrees and a radius of 2 meters both vertically and horizontally. The fifth pose is at the centre of the 4 poses. The central camera pose will be the core reference point as it will be used for the RGBD object. Furthermore, the closest point to the central camera will be used as the ground truth distance for calculating the scale factor for the sparse point cloud.

### Perform Structure from Motion on the 5 images
As soon as the 5 images are captured and present on the file-system, the structure from motion process kicks-off. Using Meshroom 2023.3.0,the pipeline considers camera initialization, Feature Extraction and Feature and Image matching and lastly Bundle adjustment to culminate the reconstruction process. The output of the SfM process are two-fold, a file containing the Camera Intrinsics as well as a sparse point cloud in the PLY format. The scale factor of the point cloud is specific to a given camera network and is always consistent regardless of the contents of the scene or target object being captured.

### Cleaning the Sparse Point Cloud
Cleaning of the Sparse Point Cloud entails removing noisy points in the reconstruction as well as parts of the background that were reconstructed. This is achieved through the **DBScan Clustering** algorithm. DBScan is able to remove the noisy points from the point cloud inherently. As for the clusters, since only one point cloud is needed, the clusters that are beyond a certain threshold distance from the centroid of the point cloud are filtered out. The rest of the clusters are then combined to form one point cloud and its normals computed.

### Extract Transformation Properties
At this point, the target point cloud is ready and is set to be "calibrated". Critical to the calibration is a second scaling factor that is applied to the target point cloud. The target point cloud has its origin as a single image which was the centroid image during the image acquisition. Thus, the closest point to the origin is calculated first and cached. As for the sparse point cloud, the origin might not be the same as the centroid's camera location. Nevertheless, the centroid camera is obtained from a file-`camera.sfm`, an output of the SfM process. The contents of the file are parsed to get the position of the centroid camera and from the obtained position, the distance between it and the closest point on the point cloud. The closest points for both point clouds should correspond, given that the reference points are the same. The two distances are then compared, and scale factor is obtained from their ration. The scale factor is then used to scale the target point cloud, in preparation of global registration. 

With the target point cloud scaled, its normals are computed as well as Fast Point Feature Histograms(FPFH) descriptors computed. The FPFH descriptors are used to obtain a transformation matrix that can be used to align the target point cloud to the sparse point cloud. With the transformation matrix and scale factor properties, the poses can now be determined whereby an RoI can be extracted, its point cloud created and transformed to create the point cloud that will be the basis of deriving poses.
