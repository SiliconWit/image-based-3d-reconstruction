# Introduction
This research is aimed at proposing a methodology for information path planning in the context of image acquisition for photogrammetry. It is to be carried out under the Mechatronic Engineering Department of Dedan Kimathi University of Technology. The research will be utilizing various simulation platforms and thus no hardware systems will be used except for computer. The simulation components to be used in the work include a Game Engine, a robotic simulation platform and a middleware representing the Path Planning Methodology itself . This chapter will seek to address in further detail the elements used in this research, as well as their role towards achieving the objectives of the research. 
# Research Design
To fulfil the objectives of the study, the elements needed are grouped into two, pre-reconstruction and Reconstruction & Evaluation. This adheres to the steps in a typical photogrammetric reconstruction workflow[[luhmannCloseRangePhotogrammetry3D2019]]. 
## Pre-Reconstruction
Pre-reconstruction refers to all processes prior to any computation by the photogrammetric reconstruction pipelines. This sub-section will outline the means by which a practical setup will be simulated to achieve the objectives of the study.

### Game Engine
Being that photogrammetry requires a collection of images of a target object or scene, and the work is based on simulation, there is need to come up with synthetic photographs. To this end, a high fidelity rendering engine has the potential to be provided this utility which will produce images that closely mimic real world photos with physical based materials[[mezheninUsingVirtualScenes2021 ,piattiVIRTUALSIMULATORPHOTOGRAMMETRY,chenSTPLS3DLargeScaleSynthetic2022]]. Furthermore, the rendering engine should provide for camera specifications that would be otherwise be utilized by the proposed methodology as well as the photogrammetric pipeline[[gajicSimulationPhotogrammetryBased3D2019]]. One critical parameter is the focal length of the camera, as well as the camera sensor dimensions. Lastly, rendering engine should also offer some form of interaction whereby the camera's extrinsic parameter may be adjusted in real-time and based on external data. This requirement pushes the requirement from a rendering engine to a game-engine. A game engine is a software that contains a collection of tools used to compile game to given platforms as well as a set of user facing tools for creating interactive 3D experiences [[toftedahlTaxonomyGameEngines]]. Some of these user facing tools relevant to this research include a graphics rendering engine, an editor, Integrated Development Environment and a Physics Engine. Unity[[UnityRealTimeDevelopment]] is the game engine of choice for this study; it has been proven to provide High Fidelity and photo realistic visuals[[guerraFlightGogglesPhotorealisticSensor2019]] 
![[FlightGoogles.png]][[guerraFlightGogglesPhotorealisticSensor2019]].

A virtual scene will be setup in Unity that satisfy favourable lighting conditions for photogrammetry [[depaolisPhotogrammetric3DReconstruction2020]]. A 3D model that represents the object to be reconstructed will also be placed within the virtual scene. The Camera component of the game engine will then be setup in a way that mimics a physical camera sensors as well as a lens. During runtime of the scene, the camera will be moved in space while being oriented towards the target object. Photos of the object will then be taken and saved to the local filesystem from where they can be accessed later as inputs to the photogrammetric reconstruction pipeline.
### Robot Simulation
In order to uphold autonomy of the proposed methodology, a robotic system has to be incorporated in order to achieve the proposed camera poses resulting from the online IPP methodology. There are plenty of tools for robot simulation tools in literature. Among them are CoppeliaSim(formerly known as VRep), Webots, Ignition, Gazebo, Mujoco, IsaacSim, PyBullet, Morse and Unity[[audonnetSystematicComparisonSimulation2022, farleyHowPickMobile2022]]. The chief requirements for this study with respect to a robotic simulation platform is that there be predefined kinematic configurations and support for interoperability. The former requirement would assume the availability of a library of robots, preferably that exist in real world setup and have tried and tested kinematic configurations. On the other hand the latter requirement calls for the ability to interface a live robotic simulation with external entities outside the robot simulator. This can be through a scripting back-end such as C/C++ or through a specialized connector to the external entity. The external entities in this case are both Unity and the IPP algorithm. [[farleyHowPickMobile2022]] compared four of the simulators with respect to a number of metrics among them being Physics Engine Support, Model Format Support, CPU Efficiency, Real time factor and Sensor Simulation Accuracy. Of the 4 simulators, CoppeliaSim[[https://www.coppeliarobotics.com/]] had the most favourable results. To that end, CoppeliaSim will be the Robotic Simulator to be used for this study.

### Path Planning Algorithm Middleware
At the heart of this research is the Information Path Planning methodology. In the course of this study, it will be referred to as the Pose Determiner(PD). Its purpose will be to propose a set of camera poses that will ensure comprehensive coverage of the target object through photographs for a complete and accurate photogrammetric 3D reconstruction. The PD will make use of an image of a preliminary viewpoint, from which information about the scene and its object of interest will be extracted. Out of the extracted information, a set of photogrammetric capture poses will be computed for the given viewpoint. The PD will then command the robotic system to move the end-effector of the Robotic system to move the camera which will in turn correspond to the movement of the camera in the Virtual Scene after which a complete cycle will have been achieved. Given that only the geometric properties of the target object will be considered, a depth map from the preliminary viewpoints will be computed to form the foundation of the utility function of the methodology. This will be made possible by Monocular Depth Estimators(MDE), which can generate depth maps out of single images.

## Reconstruction & Evaluation
This subsection highlights the setup from after the captured photos for the purpose of reconstruction have been stored on the local file system up to validation of the study.
### Reconstruction
The reconstruction process involves computing the spatial relationships of various viewpoints from a collection of images to come up with a three-dimensional representation of the subjects in the images. A 3D reconstruction pipeline in the context of photogrammetry is a set of algorithms that work together to realize a 3D representation from a collection of two-dimensional viewpoints[[OpensourceImagebased3D]]. Some of these pipelines have been packaged as complete software packages thereby democratising photogrammetric procedures to the ordinary users. One such software package is Meshroom which provides an intuitive user interface for its users and uses the AliceVision framework as its pipeline for photogrammetry[[griwodzAliceVisionMeshroomOpensource2021]]. The output from Meshroom for a typical photogrammetric process is a mesh with the file wavefront(.obj) file format. This format allows for not only geometric representation but also textures. 
### Evaluation
The evaluation process involves comparing the results of the 3D reconstruction with a Ground Truth. The metrics used to compare the quality of reconstruction is the Euclidean Distance between two corresponding points on the respective 3D representation[[OpensourceImagebased3D]]. This Euclidean distance can be referred to as the error between the Ground Truth and the reconstruction. A lower error value is desired, and thus lower value indicate better reconstructions. However, before the Euclidean distance is computed, registration of the two 3D representations have to be computed. Registration is the process of aligning two 3D representations together. Cloud Compare[[https://www.cloudcompare.org/]]
is a software tool with both registration and comparison capabilities. With CloudCompare, both models, the Ground Truth and reconstruction will be loaded in, a coarse then fine registration performed and then the Mesh-Mesh distance calculated for the registered meshes with the Ground Truth model as the reference. CloudCompare provides an aggregated value of all the Euclidean distance across the reconstruction as the Root Mean Square error(RMSe).
